# audiolib/source/mpu401.h

## File Purpose
Header file defining the interface to an MPU-401 MIDI hardware device controller. The MPU-401 was a standard MIDI interface on PC sound cards in the 1990s. Provides constants, error codes, and function prototypes for initializing the device, transmitting MIDI events, and managing real-time memory access.

## Core Responsibilities
- Define MPU-401 hardware constants (default I/O address, command codes, status flags)
- Declare error/status codes for device operations
- Export device initialization and reset functions
- Provide low-level MIDI transmission (raw bytes and structured MIDI events)
- Export memory-locking utilities for real-time MIDI interrupt safety

## Key Functions / Methods

### MPU_Init
- Signature: `int MPU_Init(int addr)`
- Purpose: Initialize and detect MPU-401 device at specified I/O address
- Inputs: `addr` – I/O port base address
- Outputs/Return: Status code (MPU_Ok on success; MPU_NotFound if device absent)
- Side effects: Sets global `MPU_BaseAddr`; performs hardware handshake
- Calls: Not inferable from header
- Notes: Should be called once at startup before other MPU functions

### MPU_Reset
- Signature: `int MPU_Reset(void)`
- Purpose: Send reset command to device and verify acknowledgment
- Inputs: None
- Outputs/Return: Status code
- Side effects: Resets device state; blocks polling device status register
- Calls: Not inferable from header

### MPU_EnterUART
- Signature: `int MPU_EnterUART(void)`
- Purpose: Switch device from intelligent mode to raw UART mode for direct MIDI byte transmission
- Inputs: None
- Outputs/Return: Status code (MPU_UARTFailed on failure)
- Side effects: Changes device operation mode; sets device state
- Calls: Not inferable from header

### MPU_SendCommand
- Signature: `void MPU_SendCommand(int data)`
- Purpose: Send a control command byte to the device
- Inputs: `data` – Command byte (e.g., MPU_CmdReset, MPU_CmdEnterUART)
- Outputs/Return: None
- Side effects: Writes to I/O port; polls `MPU_ReadyToWrite` status flag
- Calls: Not inferable from header

### MPU_SendMidi
- Signature: `void MPU_SendMidi(int data)`
- Purpose: Transmit a raw MIDI data byte to the device (used in UART mode)
- Inputs: `data` – MIDI byte
- Outputs/Return: None
- Side effects: Writes to I/O port
- Calls: Not inferable from header

**Notes on MIDI Event Functions:** `MPU_NoteOn`, `MPU_NoteOff`, `MPU_ControlChange`, `MPU_ProgramChange`, `MPU_PitchBend`, `MPU_PolyAftertouch`, `MPU_ChannelAftertouch` are high-level MIDI message wrappers. `MPU_ResetMidi` sends an all-notes-off equivalent. `MPU_LockMemory` / `MPU_UnlockMemory` manage DPMI page-locking for real-time interrupt safety.

## Control Flow Notes
Typical initialization: `MPU_Init(addr)` → `MPU_Reset()` → `MPU_EnterUART()` → `MPU_LockMemory()`. Then MIDI event functions are called in real-time. `MPU_UnlockMemory()` at shutdown. Direct I/O port polling drives all synchronous communication.

## External Dependencies
- Standard C integers (`int`, `unsigned`)
- DPMI (DOS Protected Mode Interface) – implied for memory locking (defined elsewhere)
- MPU-401 hardware I/O controller (typically at 0x330)
