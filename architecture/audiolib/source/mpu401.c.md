# audiolib/source/mpu401.c

## File Purpose
Low-level MIDI driver for MPU401-compatible sound cards. Provides functions to send MIDI events (note on/off, control changes, etc.) and manage device initialization, UART mode configuration, and real-time memory locking for DOS-era game audio.

## Core Responsibilities
- Send raw MIDI bytes with configurable I/O delay polling
- Construct and transmit complete MIDI channel messages (note on/off, aftertouch, control changes, program changes, pitch bend)
- Reset and detect MPU401 device; switch to UART mode
- Poll device status port for readiness before I/O operations
- Lock/unlock critical code sections into physical memory for deterministic real-time MIDI playback
- Read user configuration (MPUDELAY override) at initialization

## Key Types / Data Structures
None.

## Global / File-Static State
| Name | Type | Scope | Purpose |
|------|------|-------|---------|
| `MPU_BaseAddr` | `int` | extern | Base I/O port address of MPU401 device (default 0x330) |

## Key Functions / Methods

### MPU_SendMidi
- **Signature:** `void MPU_SendMidi(int data)`
- **Purpose:** Send a single MIDI data byte to the MPU401 device with busy-wait polling.
- **Inputs:** `data` — MIDI byte to transmit (0–255).
- **Outputs/Return:** None.
- **Side effects:** I/O port access; reads/writes to `MPU_BaseAddr` and `MPU_BaseAddr + 1`; busy-waits up to `MPU_Delay` iterations.
- **Calls:** `inp()`, `outp()` (conio.h macros).
- **Notes:** Polls status port (`MPU_BaseAddr + 1`) with bitmask `MPU_ReadyToWrite` (0x40) before writing to data port. Spin-loops without yielding; delay is configurable to work around slow devices.

### MPU_NoteOff
- **Signature:** `void MPU_NoteOff(int channel, int key, int velocity)`
- **Purpose:** Compose and send a MIDI Note Off event.
- **Inputs:** `channel` (0–15), `key` (note 0–127), `velocity` (0–127).
- **Outputs/Return:** None.
- **Side effects:** Calls `MPU_SendMidi()` three times; modifies device state.
- **Calls:** `MPU_SendMidi()` (3×).
- **Notes:** Transmits status byte `0x80 | channel`, then key, then velocity. No validation of ranges.

### MPU_NoteOn
- **Signature:** `void MPU_NoteOn(int channel, int key, int velocity)`
- **Purpose:** Compose and send a MIDI Note On event.
- **Inputs:** `channel` (0–15), `key` (note 0–127), `velocity` (1–127, velocity 0 implies note off).
- **Outputs/Return:** None.
- **Side effects:** Calls `MPU_SendMidi()` three times.
- **Calls:** `MPU_SendMidi()` (3×).

### MPU_SendCommand
- **Signature:** `void MPU_SendCommand(int data)`
- **Purpose:** Send a command byte to the MPU401 command port with extended polling.
- **Inputs:** `data` — command byte (e.g., `MPU_CmdReset` 0xff, `MPU_CmdEnterUART` 0x3f).
- **Outputs/Return:** None.
- **Side effects:** Writes to command port; busy-waits up to 0xffff iterations.
- **Calls:** `inp()`, `outp()`.
- **Notes:** Uses fixed loop limit 0xffff (longer than `MPU_SendMidi`). Polls same status bit as `MPU_SendMidi`.

### MPU_Reset
- **Signature:** `int MPU_Reset(void)`
- **Purpose:** Issue hardware reset command and verify acknowledgment from device.
- **Inputs:** None.
- **Outputs/Return:** `MPU_Ok` (0) on success; `MPU_NotFound` (-1) if reset fails or ACK not received.
- **Side effects:** Calls `MPU_SendCommand()`; reads data port; polls status port with bitmask `MPU_ReadyToRead` (0x80); busy-waits up to 0xffff.
- **Calls:** `MPU_SendCommand()`, `inp()`.
- **Notes:** Detects device presence by checking for `MPU_CmdAcknowledge` (0xfe) response on data port. No timeout structure; uses fixed iteration counter.

### MPU_EnterUART
- **Signature:** `int MPU_EnterUART(void)`
- **Purpose:** Switch MPU401 from intelligent mode to UART (dumb) mode for raw MIDI streaming.
- **Inputs:** None.
- **Outputs/Return:** `MPU_Ok` (0) on success; `MPU_UARTFailed` (-2) on failure.
- **Side effects:** Calls `MPU_SendCommand()`; reads device response; busy-waits up to 0xffff iterations.
- **Calls:** `MPU_SendCommand()`, `inp()`.
- **Notes:** Pattern mirrors `MPU_Reset()`. UART mode disables intelligent sequencing and allows direct MIDI byte streaming.

### MPU_Init
- **Signature:** `int MPU_Init(int addr)`
- **Purpose:** Detect and initialize MPU401 device at given I/O address; set up MIDI communication.
- **Inputs:** `addr` — base I/O port address (typically 0x330).
- **Outputs/Return:** `MPU_Ok` if UART mode entered; error code otherwise.
- **Side effects:** Sets global `MPU_BaseAddr` and `MPU_Delay` (from user parameter "MPUDELAY" or default 0x5000); calls `MPU_Reset()` up to 4 times.
- **Calls:** `USER_GetText()`, `atol()`, `MPU_Reset()`, `MPU_EnterUART()`.
- **Notes:** Retries reset up to 4 times before giving up. User parameter override allows tuning for slow devices. No range validation on addr.

### MPU_LockMemory
- **Signature:** `int MPU_LockMemory(void)`
- **Purpose:** Lock critical code section and module globals into physical memory for real-time audio safety.
- **Inputs:** None.
- **Outputs/Return:** `MPU_Ok` (0) if all locks succeed; `MPU_Error` (-1) on DPMI failure.
- **Side effects:** Calls DPMI lock functions; on failure, calls `MPU_UnlockMemory()` to clean up partial locks.
- **Calls:** `DPMI_LockMemoryRegion()`, `DPMI_Lock()`, `MPU_UnlockMemory()`.
- **Notes:** Locks `MPU_LockStart` to `MPU_LockEnd` region, plus `MPU_BaseAddr` and `MPU_Delay` globals. Prevents page faults during time-critical MIDI I/O.

### MPU_UnlockMemory
- **Signature:** `void MPU_UnlockMemory(void)`
- **Purpose:** Release physical memory locks acquired by `MPU_LockMemory()`.
- **Inputs:** None.
- **Outputs/Return:** None.
- **Side effects:** Calls DPMI unlock functions.
- **Calls:** `DPMI_UnlockMemoryRegion()`, `DPMI_Unlock()`.

## Notes
Remaining functions (`MPU_PolyAftertouch`, `MPU_ControlChange`, `MPU_ProgramChange`, `MPU_ChannelAftertouch`, `MPU_PitchBend`) follow identical patterns to `MPU_NoteOff`/`MPU_NoteOn`: each calls `MPU_SendMidi()` three times to transmit a status byte and two data bytes.

## Control Flow Notes
**Initialization phase:** `MPU_Init()` is called once at startup with device address; it internally calls `MPU_Reset()` (retry loop) and `MPU_EnterUART()` to place device in raw MIDI mode. `MPU_LockMemory()` is invoked before real-time audio starts.

**Runtime phase:** Higher-level audio code calls MIDI event functions (`MPU_NoteOn()`, `MPU_ControlChange()`, etc.), which delegate to `MPU_SendMidi()`. Each `MPU_SendMidi()` call polls the device status port and writes one byte.

**Shutdown phase:** `MPU_UnlockMemory()` is called when audio concludes.

The file declares `MPU_LockStart` and `MPU_LockEnd` markers to define the memory region containing all MIDI transmission code, ensuring deterministic latency by preventing paging faults during MIDI I/O.

- **`mpu401.h`** — Constants and extern declarations (`MPU_BaseAddr`, `MPU_Delay`, MIDI status/command macros).
